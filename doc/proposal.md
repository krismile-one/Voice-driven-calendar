# 语音日历助手 - 需求文档

## 1. 项目概述

**项目名称**: Voice Calendar Agent

**项目类型**: MVP/Demo

**开发周期**: 3天

**目标**: 实现一个以语音交互为核心的日历管理工具，支持通过语音添加、删除、查看事件提醒。

---

## 2. 核心需求

### 2.1 语音交互

| 需求项 | 说明 |
|--------|------|
| 识别语言 | 中文为主 |
| 监听模式 | 打开后持续监听，无需手动触发 |
| 离线识别 | 轻量级方案，处理简单指令（如"打开日历"、"今天有什么安排"） |
| 在线识别 | 可选接入API（如百度/讯飞），提高复杂语句识别准确率 |

### 2.2 日历功能

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 添加事件 | P0 | "帮我创建一个明天下午3点的会议" |
| 删除事件 | P0 | "取消明天的会议" |
| 查看事件 | P0 | "今天有什么安排" / "这周的日程" |
| 修改事件 | P1 | "把明天的会议改到4点" |
| 提醒通知 | P1 | 事件到达前弹窗/声音提醒 |

### 2.3 暂不支持

- 与外部日历同步（Google Calendar、Outlook等）
- 多用户/账号系统
- 移动端适配

## 3. 技术架构

### 3.1 整体架构

```
┌─────────────────┐     API请求      ┌─────────────────┐
│                 │  ──────────────> │                 │
│   前端界面      │ <──────────────  │   后端服务      │
│  (GUI/Web)      │     JSON响应     │   (FastAPI)     │
└─────────────────┘                  └─────────────────┘
        │                                      │
        │ 麦克风输入                           │ 语音识别
        ▼                                      ▼
┌─────────────────┐                  ┌─────────────────┐
│   语音采集模块  │                  │   语音识别模块  │
│  (PyAudio)      │                  │  (离线 + 在线)  │
└─────────────────┘                  └─────────────────┘
                                              │
                                              │ NLU解析
                                              ▼
                                     ┌─────────────────┐
                                     │   日历服务模块  │
                                     │  (SQLite/本地)  │
                                     └─────────────────┘
```

### 3.2 技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 包管理 | uv | 现代Python包管理器 |
| 后端框架 | FastAPI | 轻量、高性能、自动生成API文档 |
| 语音采集 | PyAudio | 跨平台音频采集 |
| 离线识别 | Vosk / Whisper | 轻量离线语音识别 |
| 在线识别 | 百度/讯飞 API | 高精度识别（可选） |
| NLU解析 | LLM API + 正则 | 大模型标准化指令 |
| 数据存储 | SQLite | 轻量本地数据库 |
| 前端（可选） | PyQt6 或 Vue + Electron | 桌面GUI或Web端 |

### 3.3 接口设计

后端提供RESTful API，前端通过HTTP调用：

```
POST   /api/events           # 添加事件
DELETE /api/events/{id}      # 删除事件
GET    /api/events           # 查看事件列表
PUT    /api/events/{id}      # 修改事件
POST   /api/voice/upload     # 上传语音，返回识别结果
GET    /api/voice/stream     # WebSocket语音流（可选）
```

## 4. 语音识别方案

### 4.1 方案对比

| 方案 | 优点 | 缺点 | 推荐场景 |
|------|------|------|----------|
| **Vosk** | 免费离线、支持中文、轻量 | 准确率一般 | 简单指令 |
| **Whisper** | OpenAI开源、准确率高 | 需要GPU、较重 | 复杂语句 |
| **百度语音API** | 准确率高、中文优化好 | 需联网、按量计费 | 生产环境 |
| **讯飞语音API** | 国内服务稳定 | 需联网、收费 | 生产环境 |

### 4.2 推荐策略

**MVP阶段**: 使用 **Vosk** 作为主要离线识别方案，满足Demo需求。

**后续扩展**: 接入百度/讯飞API作为在线识别备选，通过配置切换。
